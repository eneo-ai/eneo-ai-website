Versioner

Följ utvecklingen av Eneo

Versionsanteckningar med nya funktioner, förbättringar och buggfixar för varje release av plattformen.

Versionshistorik Kommande funktioner
v1.7.0 Senaste Se på GitHub →
Viktig förändring – kräver åtgärd! Denna version innehåller en stor databasmigrering som inte kan rullas tillbaka. Ta en snapshot eller databas-backup innan du uppdaterar. Se uppgraderingsguiden för detaljer.
Nya funktioner
  • Omarbetad modellhantering: All hantering av AI-modeller sker nu direkt i webbgränssnittet. Alla leverantörer och modeller med LiteLLM-stöd kan läggas till med några klick.
  • Modeller per organisation: Administratörer kan konfigurera vilka AI-modeller som är tillgängliga per tenant.
  • Automatiskt skapande av användare via federation: Ny valbar inställning för automatiskt skapande av OIDC-användare.
  • Tal till text i assistenter: Diktera input direkt i chatten via Web Speech API.
  • Utökade användargrupper: Enklare hantering av grupper och medlemskap direkt i gränssnittet.
  • Gruppbaserat medlemskap i delade ytor: Styr medlemskap via användargrupper.
Förbättringar
  • Bättre mobil- och appupplevelse
  • Förbättrad filhantering och nedladdning
  • SharePoint: förbättrad behörighetshantering, "Azure AD" → "Microsoft Entra ID"
  • Stabilare och mer pålitlig webbcrawling med varningar vid misslyckande
  • Stöd för vLLM och Mistral med nya ikoner och konfiguration
  • Grupperad modellväljare per leverantör
  • Förbättrad visning av AI:ns reasoning-steg
  • Uppdaterad dokumentation med nya contribution guidelines
Buggfixar
  • Fixat problem med modell-tilldelning för assistenter skapade utan modell
  • Fixat embedding-leverantörsupplösning i worker
  • Fixat OpenAPI-schemafel, krypteringsfel vid federation-maskering, Azure-modellnamn vid migrering
  • Ny felspårning med error_id för 500-svar
  • Pyright-typkontroller i CI-pipeline

Kommande

Planerade funktioner

Dessa funktioner är planerade men ännu inte släppta. Innehåll och prioritering kan ändras baserat på utveckling och användarbehov.

Omarbetad och utökad funktion för API-nycklar

API-nycklar per assistent eller space, tidsbegränsningar, specifika behörigheter (read-only/write-only) och förbättrat UI för hantering.

Funktion för driftmeddelanden

Centrala driftmeddelanden för alla användare, meddelanden i enskilda assistenter samt extern exponering för invånare och företagare.

MCP-stöd

Robust och flexibelt stöd för MCP-källor som gör att verksamheten löpande kan ansluta till nya MCP-källor över tid.
